# 睡眠音响 App PRD 工作流

## 文件约定
- PRD 文件统一存放于 `PRD\` 目录，命名格式：`睡眠音响PRD-v{版本号}-{模块名}.md`
- 设计导出图片统一存放于 `设计导出\` 目录
- 原型文件 `.pen` 存放于项目根目录
- 先骨架后血肉，逐模块细化追加

## 工作阶段
1. 阶段 A（冷启动）：无想法时提供方向选择
2. 阶段 B（需求收敛）：逐条提问确认产品方向
3. 阶段 C（概要产出）：输出页面流转图 + 功能清单
4. 阶段 D（模块细化）：逐模块展开（功能描述→交互流程→页面状态）
5. 阶段 E（UI 风格确认）：配色方案对比 → Style Guide → 图标集选定
6. 阶段 F（原型绘制）：调用 Pencil MCP 绘制页面原型 + 自校验

## 当前进度
- 阶段 A~C 已完成（v1 概要）
- 阶段 D 已完成（v2~v9 模块细化）
- 阶段 E 已重做（v11 UI 风格确认为 D 方案·Nocturne 紫，参考 Nocturne 设计系统）
- 阶段 F 待推进：首页仪表盘 → 睡眠详情 → 趋势分析（周报）

## 已知问题

### vscode_askQuestions 偶发假离线
- 现象：工具直接返回 "The user is not available to respond"，用户实际在线
- 应对：遇到此返回时重试 1 次，仍失败再降级为自主决策
