# 睡眠音响 App PRD 工作流

## 文件约定
- PRD 文件统一存放于 `PRD\` 目录，命名格式：`睡眠音响PRD-v{版本号}-{模块名}.md`
- 设计导出图片统一存放于 `设计导出\` 目录
- 原型文件 `.pen` 存放于项目根目录
- 先骨架后血肉，逐模块细化追加

## 工作阶段
1. 阶段 A（冷启动）：无想法时提供方向选择
2. 阶段 B（需求收敛）：逐条提问确认产品方向
3. 阶段 C（概要产出）：输出页面流转图 + 功能清单
4. 阶段 D（模块细化）：逐模块展开（功能描述→交互流程→页面状态）
5. 阶段 E（UI 风格确认）：配色方案对比 → Style Guide → 图标集选定
6. 阶段 F（原型绘制）：调用 Pencil MCP 绘制页面原型 + 自校验

## Pencil 操作纪律（⚠️ 不可跳过）

### 底栏固定（⚠️ layout:"vertical" 页面必检）
- layout:"vertical" 的页面，底栏位置由前面内容高度决定，内容少时底栏会漂在中间而非底部
- **固定方案**：底栏前插入弹性占位 spacer → `{type:"frame", height:"fill_container", width:"fill_container"}`
- spacer 自动撑满内容与底栏之间的剩余空间，将底栏推至页面底部
- 页面高度设为 812（标准手机屏高），与首页空态（Uq1VR，layout:"none" + 手动 y:700）效果一致
- 违例表现：底栏悬浮在内容下方而非屏幕底部

### 子页面/变体页创建（⚠️ 不可跳过）

**Pencil DSL 无复制命令，`layout:"none"` 页面结构复杂无法 DSL 手写。复制必须由用户在 Pencil UI 完成。**

| 步骤 | 谁做 | 操作 |
|------|------|------|
| 1 | 用户 | 在 Pencil 中选中父页面 → `Ctrl+D` 复制 → 拖到目标位置 → 双击名称改成变体名 |
| 2 | AI | `snapshot_layout` 找新页面 ID → `batch_get` 确认结构完整 → 仅 `Update` 差异元素 |
| 3 | AI | 截图验证 + `snapshot_layout` 查重叠 |

**禁止 AI 用 DSL 从头创建变体页**（必然缺元素、丢组件、图标变样）

- 违例表现：变体页缺底栏、缺双指标、图标回退成 emoji、字体从 Plus Jakarta Sans 变 Inter

### 全量审计
- 批量修改多个页面同一属性前，必须先做全量审计：
  1. 一次 `batch_get` 拉全部页面
  2. 归类问题 → 去重检查 → 列出修改清单
  3. 一或两轮 `batch_design` 全量修完
  4. 抽查截图验证
- 禁止逐页发现逐页修

### 共享 UI 分级处理

**reusable 组件**（纯引用，零差异）：
- 适用于所有页面完全一致的元素
- `reusable:true` 定义 → `{type:"ref", ref:"ID"}` 引用

| 组件 | ID | 说明 |
|------|-----|------|
| 状态栏 | `WwAze` | 9:41 + 电池，全宽 390px |

**模板卡片**（复制后修改）：
- 适用于结构相同但内容/状态变化的元素（底栏不同活跃态、顶导不同标题）
- Style Guide 中放模板 → 页面复制后改活跃态/标题
- 位于 Style Guide 底部，橙色边框区分

| 模板 | 说明 |
|------|------|
| 底栏导航模板 | 复制后改对应 tab 为活跃态（#22D3EE） |
| 顶部导航模板 | 复制后改标题文案 |

判断标准：**有变化部分 → 模板；完全一致 → 组件**

### 文本换行（⚠️ 严格遵守）
- 长文本需换行时：`width:"fill_container"` + `textGrowth:"fixed-width"`
- 文字自动适配父容器宽度，无需计算卡片内边距
- **禁止硬编码像素值**（如 width:319、width:280），不同卡片宽度可能不同
- 禁止手动拆成多个 text 节点模拟换行

### 每次操作后必检
1. 间距检查：`snapshot_layout` 验证无重叠
2. 页面检查：新增页面后确认与已有页面间距 80px
3. 重复检查：修改组件后确认无重复元素
4. 截图确认：每完成一组修改截图给用户确认
5. Style Guide 同步：改组件样式后同步更新
6. 变量引用：禁止硬编码 hex，统一用 `$color-xxx`

### 图标规范（⚠️ 严格遵守）
- **禁止 emoji 作为结构性图标**：Pencil 不渲染 emoji，显示为空白方块
- **必须用 SVG path 替代**：通过 Iconify API 获取 SVG path
- **工作流程**：
  1. `GET https://api.iconify.design/search?query={关键词}` 搜索图标
  2. `GET https://api.iconify.design/{prefix}/{name}.svg?width=48&height=48&color=%23{色值}` 获取 SVG
  3. 提取 `<path d="...">` 中的 geometry
  4. `Replace("emojiId", {type:"path", fill:"#色值", width:尺寸, height:尺寸, viewBox:[0,0,24,24], geometry:"..."})`
- **常用图标**：蓝牙 mdi/bluetooth、时钟 mdi/clock-outline、月亮 mdi/moon-waning-crescent、灯泡 mdi/lightbulb-outline、警告 mdi/alert-circle、庆祝 mdi/party-popper
- **新增页面必检**：创建或复制页面后，检查是否有 emoji 文本节点，如有则替换为 SVG path

### batch_design 已知问题（⚠️ 不可跳过）
- **emoji→SVG 替换失败**：Replace 对深层子节点的 emoji text 可能报 `Cannot read properties of undefined (reading 'type')`。解决方案：先 Delete 再 Insert（分两次 batch_design 调用，不可合并在同一次）
- **孤儿节点缓存**：batch_design 中途失败后，可能残留孤儿节点导致后续 Insert 报 `No parent for node`。解决方案：该页面后续操作暂停，等用户手动删除残留节点后再继续
- **layout:"none" 页面无新节点**：Insert 到 layout:"none" 页面的子节点不渲染。解决方案：必须由用户 Ctrl+D 复制父页，AI 只能 Update 差异
- **新增元素前必须读完整页面**：Insert 前必须先 `batch_get` 读取目标页面的完整子节点列表，确认无重复元素。禁止仅凭记忆或之前的截图直接 Insert，否则容易搞出重复组件
